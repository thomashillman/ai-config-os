# Cloudflare Worker Deployment Runbook

This guide explains how to deploy the ai-config-os Worker to Cloudflare staging and production environments.

## Overview

The deployment process has two parts:

1. **Repo-side:** Build and validate artifacts, then deploy to Cloudflare
2. **Cloudflare-side:** Create and configure resources (external to this repo)

This document covers the repo-side steps. Cloudflare resource setup is manual and one-time.

## What Gets Deployed

- **Worker code** (`worker/src/`) ‚Äî TypeScript sources compiled by Wrangler
- **Bundled artifacts** (`dist/`) ‚Äî Skill registry and client plugin JSON (generated by `npm run build`)
- **Configuration** (`wrangler.toml`) ‚Äî Bindings, environment variables, secrets

## Prerequisites

### On Your Machine

1. Node.js 18+ installed
2. npm dependencies installed:
   ```bash
   npm install
   cd worker && npm install
   ```
3. Cloudflare Wrangler installed and authenticated:
   ```bash
   npm install -g wrangler
   wrangler auth
   ```

### In Cloudflare (One-time Setup)

Create these resources **once** and keep the IDs/names for the steps below:

#### 1. KV Namespace for Manifest Storage

```bash
# Create staging namespace
wrangler kv:namespace create "manifest-kv" --preview false --env staging

# Create production namespace (optional)
wrangler kv:namespace create "manifest-kv" --preview false
```

Copy the namespace ID and update `wrangler.toml`:
- Under `[env.staging.kv_namespaces]`: set `id = "your-staging-kv-id"`
- Under `[kv_namespaces]`: set `id = "your-production-kv-id"` (if deploying to production)

#### 2. R2 Bucket for Artifacts

```bash
# Create staging bucket
wrangler r2 bucket create ai-config-os-artefacts-staging

# Create production bucket (optional)
wrangler r2 bucket create ai-config-os-artefacts
```

These are already referenced in `wrangler.toml`.

#### 3. Remote Executor Service

Set up a service to handle tool execution (outside Cloudflare). It must:
- Listen on an HTTP endpoint (e.g., `https://executor-staging.example.com`)
- Support `POST /v1/execute` route
- Use the same `EXECUTOR_SHARED_SECRET` as the Worker

See `runtime/remote-executor/` for the reference implementation.

#### 4. Worker Secrets

Set these via Wrangler CLI (values stored securely in Cloudflare):

```bash
# For staging deployment
wrangler secret put AUTH_TOKEN --env staging
# Paste your staging bearer token

wrangler secret put EXECUTOR_SHARED_SECRET --env staging
# Paste your shared executor secret

# Optionally set rotation token
wrangler secret put AUTH_TOKEN_NEXT --env staging
```

For production, repeat with `--env production` instead of `--env staging`.

## Deploy to Staging

### Step 1: Build Artifacts

```bash
npm run build
# Or manually:
bash scripts/deploy/build-worker.sh
```

This generates `dist/` with:
- `dist/registry/index.json` ‚Äî Skill registry with versions
- `dist/clients/claude-code/.claude-plugin/plugin.json` ‚Äî Client metadata

### Step 2: Validate Configuration

```bash
npm run deploy:validate
# Or manually:
node scripts/deploy/validate-config.mjs
```

This checks:
- `wrangler.toml` structure
- Required KV and R2 bindings
- Environment variable placeholders

### Step 3: Deploy to Staging

```bash
npm run deploy:staging
# Or manually:
bash scripts/deploy/deploy-staging.sh
```

This:
1. Runs validation
2. Checks build artifacts
3. Runs `wrangler deploy --env staging`
4. Reports Worker URL

**Output:**
```
‚úì Deployment successful!

Next steps:
  1. Note your Worker URL (e.g., https://ai-config-os.your-domain.workers.dev)
  2. Run smoke tests:
     export AI_CONFIG_WORKER_URL="https://your-worker-url/v1"
     export AI_CONFIG_WORKER_TOKEN="your-staging-token"
     node scripts/deploy/smoke-tests.mjs
```

### Step 4: Run Smoke Tests

```bash
export AI_CONFIG_WORKER_URL="https://your-staging-worker.workers.dev/v1"
export AI_CONFIG_WORKER_TOKEN="your-staging-token"

npm run smoke:test
# Or manually:
node scripts/deploy/smoke-tests.mjs
```

Tests verify:
- ‚úì `GET /health` ‚Äî Worker responds
- ‚úì `GET /manifest/latest` ‚Äî Registry is accessible (or falls back)
- ‚úì `GET /client/claude-code/latest` ‚Äî Client plugin works
- ‚úì `POST /tasks` ‚Äî Task creation succeeds
- ‚úì `POST /execute` ‚Äî Executor communication works (or fails gracefully)
- ‚úì Auth required ‚Äî `401` when no token provided

**Success output:**
```
Smoke Tests ‚Äî Deployment Readiness Check
=========================================

GET /health ‚úì 200 - OK
GET /manifest/latest ‚úì 200 - OK
GET /client/claude-code/latest ‚úì 200 - OK
POST /tasks ‚úì 201 - OK
POST /execute ‚úì 502 (executor unavailable but auth works)
Auth Requirement Tests
GET /health (no auth) ‚úì 401 - Auth required

6 passed, 0 failed

‚úì All smoke tests passed. Deployment is ready.
```

## Deploy to Production (Optional)

Once staging is validated, deploy to production:

### Step 1: Set Production Secrets

```bash
wrangler secret put AUTH_TOKEN
# Paste your production token

wrangler secret put EXECUTOR_SHARED_SECRET
# Paste your production secret
```

### Step 2: Deploy

```bash
npm run build
wrangler deploy --config worker/wrangler.toml
```

**Note:** This uses the default `[vars]` section, not `[env.staging]`.

### Step 3: Validate

```bash
export AI_CONFIG_WORKER_URL="https://your-production-worker.workers.dev/v1"
export AI_CONFIG_WORKER_TOKEN="your-production-token"
npm run smoke:test
```

## Local Development

### Run Worker Locally

```bash
cd worker
wrangler dev
# Worker available at http://localhost:8787
```

To load local env vars, create `worker/.env.local`:
```
AUTH_TOKEN=local-test-token
EXECUTOR_SHARED_SECRET=local-test-secret
EXECUTOR_PROXY_URL=http://localhost:8788
```

### Start Local Executor (in another terminal)

```bash
export REMOTE_EXECUTOR_PORT=8788
export REMOTE_EXECUTOR_SHARED_SECRET="local-test-secret"
bash scripts/deploy/start-local-executor.sh
```

### Test Locally

```bash
# Set local endpoints
export AI_CONFIG_WORKER_URL="http://localhost:8787/v1"
export AI_CONFIG_WORKER_TOKEN="local-test-token"

# Run smoke tests
npm run smoke:test
```

## Environment Variables Summary

### Worker Secrets (set via `wrangler secret put`)

| Name | Required | Purpose |
|------|----------|---------|
| `AUTH_TOKEN` | Yes | Bearer token for API authentication |
| `AUTH_TOKEN_NEXT` | No | Next token for rotation |
| `EXECUTOR_SHARED_SECRET` | Yes | Shared secret for executor verification |
| `HANDOFF_TOKEN_SIGNING_KEY` | No | Private key for handoff token signing |

### Worker Public Variables (in `wrangler.toml`)

| Name | Required | Purpose |
|------|----------|---------|
| `ENVIRONMENT` | Yes | Environment label: `staging`, `production` |
| `EXECUTOR_PROXY_URL` | Yes | URL to remote executor (e.g., `https://executor.example.com`) |
| `EXECUTOR_TIMEOUT_MS` | No | Executor timeout in ms (default: 10000) |

### Cloud Resources (in `wrangler.toml`)

| Binding | Type | Purpose |
|---------|------|---------|
| `MANIFEST_KV` | KV Namespace | Stores version pointers and manifests |
| `ARTEFACTS_R2` | R2 Bucket | Stores skill artifacts (JSON files) |

### Executor Environment (on executor service)

| Name | Required | Purpose |
|------|----------|---------|
| `REMOTE_EXECUTOR_PORT` | No | Port to listen on (default: 8788) |
| `REMOTE_EXECUTOR_SHARED_SECRET` | Yes | Must match worker's `EXECUTOR_SHARED_SECRET` |
| `REMOTE_EXECUTOR_TIMEOUT_MS` | No | Tool execution timeout (default: 15000) |
| `REMOTE_EXECUTOR_SIGNATURE_PUBLIC_KEY_PEM` | No | Public key for request signature verification |
| `REMOTE_EXECUTOR_REQUIRE_SIGNATURE` | No | Require request signatures if set to `'true'` |

## Troubleshooting

### "Missing KV namespace ID" or "Missing R2 bucket"

**Problem:** Validation fails because `wrangler.toml` has placeholder values.

**Solution:**
1. Create KV namespace and R2 bucket in Cloudflare (see Prerequisites)
2. Update `wrangler.toml` with real IDs and bucket names
3. Rerun `npm run deploy:staging`

### "Unauthorized (401)" in Smoke Tests

**Problem:** Requests fail with 401 even with correct token.

**Solution:**
1. Check token matches what you set: `wrangler secret list --env staging`
2. Verify token is not expired or rotated
3. Check `Authorization: Bearer <token>` header in request

### "Executor unavailable (502/504)" in Smoke Tests

**Problem:** Executor connectivity fails.

**Solution:**
1. Verify `EXECUTOR_PROXY_URL` in `wrangler.toml` is correct and reachable
2. Check executor service is running: `bash scripts/deploy/start-local-executor.sh`
3. Verify `EXECUTOR_SHARED_SECRET` matches on both Worker and executor
4. Check executor logs for errors

### Build Fails: "dist/ not created"

**Problem:** `npm run build` completes but `dist/` is missing.

**Solution:**
1. Check for errors in build output
2. Verify skill files are valid: `npm run validate`
3. Run with verbose output: `node scripts/build/compile.mjs`
4. Check disk space and permissions

## What Remains Manual

The following **must** be done outside the repo:

1. ‚úì Create Cloudflare KV and R2 resources (one-time)
2. ‚úì Update `wrangler.toml` with resource IDs
3. ‚úì Set secrets via `wrangler secret put` (one-time per environment)
4. ‚úì Set up executor service (your infrastructure)
5. ‚úì Verify executor endpoint is reachable from Cloudflare
6. ‚úì Set up custom domain (optional; Cloudflare provides default)

Everything else ‚Äî build, validation, and deployment ‚Äî is automated.
